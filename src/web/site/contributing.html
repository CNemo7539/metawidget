Contributing

Metawidget is developed by the community. You can contribute by:

  • providing feedback on the forums
  • reporting issues
  • adding to the Wiki
  • contributing enhancements to the code and documentation

For the latter, submit a patch either via the issue tracker or by sending it to support@metawidget.org. Full source code and documentation is included in the download. Some starting points in the documentation include:

  • Architecture overview
  • Implementing your own Inspector
  • Implementing your own WidgetBuilder

You can also browse the source code online, such as source code for the supported front-ends and source code for the supported back-ends.

Some enhancements suggested by users to date include (in alphabetical order):

Suggestion Description
Flex support Flex is a popular RIA platform and has interesting architectual differences compared to our currently supported Web frameworks. In particular, it is not Java-based.
Maven support Metawidget integrates with lots of third-party libraries, each of which is continually being updated by their respective owners. Adding Maven support would help other contributors maintain version dependencies.
ZK support ZK is a very popular Web framework, and has interesting architectual differences compared to our currently supported Web frameworks.
