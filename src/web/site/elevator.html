Elevator Pitch

The Metawidget 'elevator pitch' gives a quick overview of Metawidget in the time it takes to ride an elevator:

Morning John
Morning Bob
Hey, so what's that Metawidget stuff you've been working with, Bob?
Metawidget is UI generation done right. It believes UI generation should be an everyday part of the software stack, but that current approaches don't cut it
How come?
3 reasons: first, static code generation is a no go. It's great for giving you a headstart, but then you need to tweak the generated code, and regeneration gets cumbersome
Static generation is like a friend that gets you started on the road, but can't stay for the journey
Exactly. UI generation needs to be done at runtime
Second, UI generation shouldn't 'own' the UI - you can't expect to generate everything. UIs are full of asthetics and psychology. The pieces you can automate have to fit nicely alongside other screen elements
Finally, UI generation should be a regular part of the software stack. It should work with your choice of persistence layer, business layer, UI framework, UI component libraries...
...not dictate them
Exactly
Well, that sounds cool. Maybe I should give it a try
It's been working great for us. Let me know how it goes for you. See you later John
