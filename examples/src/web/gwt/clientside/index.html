Take a walk on the client side...

By default, GwtMetawidget inspects business objects server-side. This is because client-side JavaScript does not support reflections or annotations.

However if you don't need reflections or annotations, and have your own way of retrieving inspection results, you can plug in your own Inspector and keep everything client-side. This example implements a TextAreaInspector that retrieves inspection results from the textarea (on the left), and generates the UI (on the right).

Inspection Results

Edit the inspection results here, or use the buttons to load some samples. Then click Generate to see the UI. Note the inspection result must be well-formed.

Generated Metawidget

The UI is generated here using pure client-side JavaScript. Click the Save button to display the values retrieved from the UI
